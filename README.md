# Telegram News Bot (MVP)

MVP-бот для Telegram:
- забирает новости из RSS,
- сначала отправляет администратору сокращенный текст новости из RSS с источником,
- после подтверждения пытается извлечь полный текст статьи по ссылке (с fallback на RSS summary) и делает короткий профессиональный перевод/саммари на русский через LLM,
- отправляет итоговый текст администратору на финальную модерацию,
- публикует в канал после подтверждения (в тестовом режиме — в отдельный тестовый канал).

## Логика
1. Бот периодически опрашивает RSS.
2. Для новых ссылок отправляет админу карточку с заголовком, кратким текстом из фида и источником:
- `Принять в работу`
- `Отклонить`
3. Если админ нажал `Принять в работу`, бот генерирует итоговый текст на русском без указания источника и отправляет на финальное подтверждение:
- `Опубликовать`
- `Переписать`
- `Отклонить`
4. Действия финального этапа:
- `Опубликовать`: пост в канал.
- `Переписать`: предыдущее предложение удаляется, генерируется новый вариант.
- `Отклонить`: предложение удаляется, публикации не будет.
- Альтернатива `Опубликовать`: админ может ответить (цитированием) на карточку финального этапа своим текстом, и в канал уйдет именно этот текст.

## Ограничения MVP
- Источник: один RSS (можно добавить через `RSS_FEEDS`).
- Дедупликация между разными источниками не реализована как отдельный модуль, но есть защита от повторной обработки ссылки.
- Провайдер LLM: `openai` (архитектурно вынесен в настройку `LLM_PROVIDER` для расширения).

## Тестовый режим публикации
- Запуск `make run-test` включает тестовый режим публикации.
- Для Docker используйте `make docker-up-test`.
- В тестовом режиме публикация идет в `TELEGRAM_TEST_CHANNEL_ID`.
- Обычный запуск `make run`/`make docker-up` публикует в `TELEGRAM_CHANNEL_ID`.

## Команды Бота
- `/start` — показать список команд.
- `/check` — вручную запустить проверку RSS.
- `/stats` — показать текущее состояние бота.
- `/clean` — очистить `pending` (карточки на модерации), не трогая `seen_links`.

## Поля В /stats
- `pending` — количество карточек, которые ждут действия админа.
- `seen_links` — количество ссылок, уже обработанных ботом (защита от дублей).
- `admin_chat_blocked` — `true`, если `TELEGRAM_ADMIN_CHAT_ID` настроен неверно или недоступен.
- `channel_blocked` — `true`, если бот не может писать в целевой канал публикации.
- `publish_mode` — текущий режим публикации: `prod` или `test`.
- `publish_channel_var` — какая env-переменная сейчас используется для канала.
- `publish_channel_id` — id текущего канала публикации.
- `publish_channel_name` — название (или `@username`) текущего канала публикации.

## Файлы
- `src/bot.py` — основной бот и бизнес-логика
- `src/config.py` — загрузка настроек
- `src/llm.py` — интеграция с LLM
- `src/storage.py` — локальное состояние (`seen_links`, `pending`)
